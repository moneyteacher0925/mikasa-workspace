# 대규모 조직을 위한 1Password 설정 가이드

수천 명의 직원, 여러 사무실, 또는 복잡한 팀 구조를 가진 조직이라면, 이 가이드를 참고하여 1Password가 기업 모델을 어떻게 지원할 수 있는지 확인하세요.

---

## 용어 정리

### 소유자 (Owner)
소유자는 모든 관리 권한을 가지며, 결제 관리, 그룹/볼트 관리, 팀원 추가/제거, 계정 복구를 수행할 수 있습니다. 각 팀에는 기본적으로 한 명의 소유자(팀을 생성한 사람)가 있지만, 백업을 위해 추가 소유자를 지정해야 합니다.

### 관리자 (Administrator)
관리자는 볼트를 생성/보관하고, 볼트 권한을 관리하며, 팀에 다른 사람을 초대하고, 계정 복구를 수행할 수 있습니다. 팀 생성자는 기본적으로 관리자 그룹에 포함됩니다. 추가 팀원을 관리자 그룹에 추가할 수 있습니다.

### 사이트 관리자 (Site Administrator)
조직이 여러 지역에서 운영되는 경우, 사이트 관리자를 지정할 수 있습니다. 이들은 1Password에서 관리자 권한을 가지지만, 해당 지역의 직원만 관리합니다.

### 그룹 (Groups)
1Password에는 팀원에게 관리 책임을 위임할 수 있는 3개의 기본 그룹이 있습니다. 팀에서 기능은 그룹 멤버십에 따라 달라집니다. 1Password Business에서는 커스텀 그룹도 사용할 수 있습니다.

### 볼트 (Vaults)
볼트를 통해 데이터를 정리하고 누가 어떤 데이터에 접근할 수 있는지 제어할 수 있습니다. 볼트는 일부 또는 전체 팀원과 공유할 수 있으며, 다양한 대상과 목적에 맞게 생성할 수 있습니다.

---

## 분산형 조직에서 1Password Business 설정 방법

많은 조직이 완전히 독립적으로 운영되는 부서로 나뉘어 있습니다. 지역별, 업무 분야별, 또는 기타 기준으로 분리될 수 있습니다. 이런 모델을 사용하는 팀이라면, 1Password도 동일한 방식으로 접근하여 각 단위에 사이트 관리자를 지정해야 합니다.

### 예시 사용 사례

5개의 서로 다른 지역 사업부가 1Password를 사용하고 있으며, 각 사업부에 관리자를 두고 싶은 경우:

기본 관리자 그룹을 사용하는 대신, 각 사업부별로 별도의 커스텀 그룹을 생성합니다:

| 그룹 | 구성 |
|------|------|
| 소유자 | (전체 관리) |
| 북미 사이트 관리자 | 볼트 + 그룹 관리 |
| 유럽 사이트 관리자 | 볼트 + 그룹 관리 |
| 아시아태평양 사이트 관리자 | 볼트 + 그룹 관리 |
| 중남미 사이트 관리자 | 볼트 + 그룹 관리 |

---

## 워크플로우 단계

1. 각 사업부별 **커스텀 그룹을 생성**하고, 선정된 사이트 관리자를 추가합니다.

2. 사이트 관리자에게 **원하는 수준의 권한**을 부여합니다. 최소한 **볼트 생성(Create Vaults)**과 **복구(Recovery)** 권한을 포함하세요.

3. **볼트 생성 워크플로우**를 직접 개발하거나, 아래의 일반적인 워크플로우를 따르세요:
   - 최종 사용자가 볼트 생성을 요청합니다 (이메일, 티켓 서비스 등을 통해)
   - 사이트 관리자가 요청을 받고 볼트 생성에 필요한 세부 정보를 수집합니다 (예: 볼트 이름, 접근 권한을 가질 사람 등)
   - 볼트가 생성되면, 사이트 관리자는 해당 사업부의 **사이트 관리자 커스텀 그룹을 추가**하고 **관리(Manage) 권한**이 켜져 있는지 확인합니다. 이를 통해 사이트 관리자가 이후에도 볼트를 변경할 수 있습니다.
   - 볼트를 생성한 사이트 관리자는 볼트에서 **개인 사용자로서 자신을 제거**해야 합니다. 계정의 상위 사이트 관리자 그룹에 속해 있으므로, 모든 볼트를 관리할 수 있는 관리자 권한은 유지됩니다.
   - 나머지 사용자 또는 그룹을 볼트에 추가하고 원하는 권한을 설정합니다.
