2026.03.19 (목) 17:00 | 구글밋 | 천주원 개발자 × 에렌 대표

미팅 개요

안건딱모여(플로우머스) 웹사이트 개발 방향 논의
참석천주원 (개발자) / 권에렌 (플로우머스 공동대표)
공유 대상오맷돌 대표님 (플로우머스 공동대표)

핵심 결론

처음부터 자체 개발하지 않고, 기존 플랫폼으로 검증 → 기존 솔루션 활용 → 자체 개발 3단계로 진행. 어드민(관리자 페이지)을 처음부터 만들면 비용·시간이 과다하므로, 기존 어드민이 있는 솔루션을 먼저 활용하는 것을 권장.

개발 로드맵 (3단계)

1기존 플랫폼에서 모임 테스트

문토 / 밋업에 호스트로 직접 들어가서 모임 운영

  • 어떤 카테고리가 반응 좋은지 데이터 확보
  • 노쇼율, 재참여율, 참가자 피드백 수집
  • 모임 운영 프로세스 정립
  • 비용: 거의 0원

맷돌 호스트 운영 + 현장 영상/사진 촬영
에렌 데이터 분석 + 광고 테스트

2기존 솔루션(어드민) 활용

라이브클래스 등 기존 어드민이 갖춰진 플랫폼 활용

  • 회원관리 / 결제 / 일정관리 어드민이 이미 있음
  • 커스터마이징만 하면 빠르고 저렴
  • 모임 운영에 필요한 핵심 기능 커버 가능
  • 1단계에서 검증된 데이터 기반으로 필요 기능 선별

천주원 솔루션 세팅 + 커스터마이징
에렌 요구사항 정리 + 운영

3자체 웹사이트 개발

트래픽과 수익이 검증된 후 자체 플랫폼 개발

  • 맷돌 바이브코딩으로 프론트엔드 와꾸 잡기 (모든 페이지 + 어드민)
  • 천주원 프론트 기반으로 백엔드 개발 → 비용 절감
  • 1~2단계에서 파악한 "진짜 필요한 기능"만 개발
  • 불필요한 기능 제거 → 시간·비용 최소화

이 순서가 좋은 이유

항목처음부터 자체개발3단계 진행 (권장)
개발 기간3~6개월+1단계 즉시 시작
초기 비용수천만원거의 0원
리스크만들고 피벗하면 전부 폐기검증 후 개발 → 낭비 없음
어드민직접 개발 (가장 복잡)기존 솔루션 활용 → 나중에 자체 개발
사업 검증개발 끝나야 시작개발 전에 이미 검증 완료
