  - 금~화: 거래처 입고 + 물류팀 작업
  - 수 12시: 물류팀 출고가능 수량/파렛트 보고 → 14시 밀크런 접수 마감 (2시간)
  - 목 오후: 밀크런 물류서류 출력/부착
  - 금 오전: 밀크런 출고
- 작업불가 SKU 의사결정 플로우: 쿠팡 내부재고 확인 → 충분하면 발주서 통째 미루기, 부족하면 마진율 기준 결정
- 발주서 단위로만 미루기 가능 (SKU별 개별 미루기 불가)
- 3가지 개선안: SKU별 리스트 템플릿, 화요일 입고마감 체크 신설, 전달기록(카톡/메일)

## 쿠팡 발주 파일 분석 (아르민 파일 3개 수신)
- **파일1 (CSV)**: 쿠팡 서플라이어허브 원본 발주서 — 277행, 23열, 발주번호 10개, 매입가/총발주매입금 포함
- **파일2 (XLSX, PO_SKU_LIST)**: 289행, 8열 — 역할 아직 미확인 (아르민에게 질문함)
- **파일3 (XLSM, 매크로)**: 아르민이 만든 SKU별 발주리스트
  - 시트1 "재고파악양식": SKU명(=거래처그룹), SKU ID, SKU이름, 입고예정일, 발주수량, 현재수량(빈칸), 필요수량(빈칸) — 275행
  - 시트2 "SKU기본정보": SKU ID→SKU이름→발주처 매핑 373개, 거래처 35개
- 파일 역할 정리:
  - 쿠팡원본(CSV) = 발주 수신 시 다운로드
  - SKU별 발주리스트(XLSM) = 물류팀 출고작업용 (밀크런 출고까지 계속 사용)
  - 거래처별 발주리스트 = 발주 시 일회성 사용 (물류팀장이 재고파악 후 거래처에 발주요청)
- 거래처 그룹: 자체제작(소포장/중포장), 그린랜드, 김해삼계마사, 남영플라스틱, 봄찬가드닝, 중국공장 다수 등
- 복합거래처 존재: "김해삼계마사+가든잇_중포장(자체제작)" 등
